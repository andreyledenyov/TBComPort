О компоненте

Компонент для обмена данными компьютера с внешними устройствами через интерфейс RS-232 в синхронном режиме. Работает с Delphi 3..10, 2007..2010 и C++ Builder 3..6. Подробно о свойствах, методах и событиях компонента TBComPort см. в файле справки BComPort.chm. Каталог Demo содержит демонстрационные примеры использования компонента.

Компонент разработан основе идей одной из лучших свободно-распостраняемых библиотек ComPort Library от Dejan Crnila.

Установка

  • Delphi
    • Файлы компонента (BCPort.pas, BCPort.inc и BCPort.dcr) поместите в каталог "Imports" среды программирования (например, "C:\Program Files\Borland\Delphi5\Imports")
    • Запустите Delphi и в меню "Component" щелкните команду "Install Component..."
    • В появившемся диалоге "Install Component" перейдите на вкладку "Into new package". В поле "Unit file name:" с помощью кнопки "Browse..." введите полное (с путем к нему) имя файла BCPort.pas, ранее помещенного в каталог "Imports". В полях "Package file name:" и "Package description:" введите "BComPort". Щелкните кнопку "ОК".
    • В диалоге "Confirm" щелкните кнопку "Yes".
    • Закройте Delphi. При запросе сохранения изменений ответьте "Yes".

      В палитре компонентов на вкладке Samples появится компонент.

  • C++ Builder

    Честно говоря, не программирую на C++, поэтому доверьтесь в вопросе установки литературе и знающим людям... :)

История версий

2.20 (01.01.2012)

Компонент существенно переработан...

  • Расширен диапазон значений свойства BaudRate для применения компонента в системах с преобразователями USB-RS232 на высоких скоростях.
  • Добавлено свойство CustomBaudRate для установки значений скоростей обмена, не совпадающих со стандартным рядом скоростей.
  • Добавлен объект DataPacket, позволяющий автоматизировать выделение из входящего потока данных определенных фрагментов (пакетов). Компонент оснащен соответствующим событием OnPacket.
  • Добавлена обработка состояния "разрыва связи": метод SetBreak и событие OnBreak.
  • Управление приоритетом потока мониторинга событий теперь является published-свойством (доступно на этапе конструирования формы).
  • Процедура EnumComPorts теперь не дополнительная, а метод компонента.
  • Исключены свойства и методы асинхронных операций как не имеющие практического значения.
  • Добавлен файл BCPort.inc для обеспечения совместимости с указанными выше версиями Delphi и C++ Builder.
  • Переработаны демонстрационные примеры и справка.
